## 1. 系统概述

### 1.1 系统简介

用户管理系统是一个基于 Web 的全栈应用，采用分层架构设计，提供用户注册、登录、权限管理等核心功能。系统分为前端（HTML模板）、后端（Go HTTP服务）和数据库（MySQL）三个层次。

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| 用户注册 | 支持用户名、密码、邮箱注册 |
| 用户登录 | 基于 JWT 的身份认证 |
| 用户管理 | 管理员可管理所有用户 |
| 权限控制 | 基于 RBAC 的角色权限系统 |
| 密码加密 | 防止明文存储（待实现 bcrypt） |

### 1.3 技术特点

- **轻量级**：使用 Go 标准库，无复杂依赖
- **高性能**：Go 的并发处理能力
- **可扩展**：分层架构，易于功能扩展
- **安全性**：JWT 认证、CORS 控制、SQL 注入防护

## 2. 技术架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                        前端层                           │
│  HTML5 Template + JavaScript                            │
└─────────────────────────────────────────────────────────┘
                            ↕ HTTP
┌─────────────────────────────────────────────────────────┐
│                       Handler 层                         │
│  HTTP 请求处理、参数解析、响应转换                        │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                       Service 层                         │
│  业务逻辑、权限校验、数据计算                            │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                      Repository 层                      │
│  数据库 CRUD 操作、SQL 执行、数据映射                    │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                       数据层                             │
│  MySQL 数据库                                           │
└─────────────────────────────────────────────────────────┘
```

### 2.2 架构原则

#### 分层原则
- **Handler 层**：只负责 HTTP 协议处理，不包含业务逻辑
- **Service 层**：只负责业务逻辑，不直接操作数据库
- **Repository 层**：只负责数据访问，不包含业务规则

#### 单一职责原则
- 每层只专注于自己的职责
- 层与层之间通过接口通信
- 避免循环依赖

## 4. 技术实现详解

### 4.1 Handler 层

#### 职责
- 接收 HTTP 请求
- 解析请求参数
- 调用 Service 层
- 返回 HTTP 响应

#### 代码示例

```go
// user_handler.go
func RegisterHandler(w http.ResponseWriter, r *http.Request) {
    // 1. 只接受 POST 请求
    if r.Method != http.MethodPost {
        w.WriteHeader(http.StatusMethodNotAllowed)
        return
    }

    // 2. 解析表单数据
    if err := r.ParseForm(); err != nil {
        w.WriteHeader(http.StatusBadRequest)
        return
    }

    username := r.FormValue("username")
    password := r.FormValue("password")
    email := r.FormValue("email")

    // 3. 调用 Service 层处理业务
    user, err := UserSvc.Register(username, password, email)
    if err != nil {
        w.WriteHeader(http.StatusBadRequest)
        fmt.Fprintln(w, "注册失败:", err)
        return
    }

    // 4. 返回响应
    fmt.Fprintf(w, "注册成功!用户名 : %s,用户ID: %d\n", user.Username, user.ID)
}
```

#### 路由注册

```go
// user_handler.go
func RegisterUserRoutes(mux *http.ServeMux, svc *service.UserService) {
    // restful 风格路由
    mux.HandleFunc("/users", LoginHandler)           // 注册（POST）
    mux.HandleFunc("/users/", RegisterHandler)        // 用户管理（GET/PUT/DELETE）

    // 静态文件路由
    mux.HandleFunc("/login", func(w http.ResponseWriter, r *http.Request) {
        http.ServeFile(w, r, "web/templates/login.html")
    })
}
```

### 4.2 Service 层

#### 职责
- 业务规则验证
- 复杂业务流程编排
- 调用 Repository 层

#### 代码示例

```go
// user_service.go
type UserService struct {
    Repo *repository.UserRepository
}

// 注册函数
func (s *UserService) Register(username, password, email string) (*model.User, error) {
    // 1. 验证用户名格式
    if isValid, err := s.IsValidUsername(username); !isValid {
        return nil, err
    }

    // 2. 检查用户名是否已存在
    if exists, _ := s.IsExistUsername(username); exists {
        return nil, errors.New("用户名已存在")
    }

    // 3. 验证密码格式
    if isValid, err := s.IsValidPassword(password); !isValid {
        return nil, err
    }

    // 4. 创建用户
    user := &model.User{
        Username: username,
        Password: password,  // 注意：实际应用应加密
        Email:    email,
        Role:     "user",
        Status:   1,
    }

    return s.Repo.CreateUser(user)
}

// 用户名格式验证
func (s *UserService) IsValidUsername(username string) (bool, error) {
    // 长度校验：6-13个字符
    if len(username) < 6 || len(username) > 13 {
        return false, errors.New("用户名长度不能小于6位,不能大于13位")
    }

    // 格式校验：仅字母和数字
    if alphaNumRegex.MatchString(username) {
        return false, errors.New("用户名格式错误,只能由字母和数字组合")
    }

    return true, nil
}
```

#### 性能优化

```go
// 使用预编译正则表达式，避免每次调用都重新编译
var alphaNumRegex = regexp.MustCompile(`^[a-zA-Z0-9]+$`)

// IsValidUsername
func (s *UserService) IsValidUsername(username string) (bool, error) {
    // 直接使用预编译的正则表达式
    if alphaNumRegex.MatchString(username) {
        return false, errors.New("用户名格式错误")
    }
    return true, nil
}
```

### 4.3 Repository 层

#### 职责
- 数据库 CRUD 操作
- SQL 查询构建
- 数据映射（ORM）

#### 代码示例

```go
// user_repository.go
type UserRepository struct {
    DB *sql.DB
}

// 创建用户
func (r *UserRepository) CreateUser(user *model.User) (int64, error) {
    query := `
        INSERT INTO users(username, password, email, role, status)
        VALUES (?, ?, ?, ?, ?);
    `

    result, err := r.DB.Exec(query,
        user.Username,
        user.Password,
        user.Email,
        user.Role,
        user.Status,
    )

    if err != nil {
        return 0, err
    }

    return result.LastInsertId()
}

// 根据用户名查询用户
func (r *UserRepository) GetUserByUsername(username string) (*model.User, error) {
    query := `
        SELECT id, username, password, email, role, status,
               created_at, updated_at
        FROM users
        WHERE username = ?;
    `

    row := r.DB.QueryRow(query, username)

    var user model.User
    err := row.Scan(
        &user.ID,
        &user.Username,
        &user.Password,
        &user.Email,
        &user.Role,
        &user.Status,
        &user.CreatedAt,
        &user.UpdateAt,
    )

    if err != nil {
        return nil, err
    }

    return &user, nil
}
```

#### 最佳实践

1. **使用参数化查询**：防止 SQL 注入
   ```go
   // ✅ 正确：使用占位符
   r.DB.Exec(query, username, password)

   // ❌ 错误：字符串拼接（存在 SQL 注入风险）
   r.DB.Exec("SELECT * FROM users WHERE username = '" + username + "'")
   ```

2. **设置连接池参数**
   ```go
   DB.SetMaxOpenConns(25)    // 最大打开连接数
   DB.SetMaxIdleConns(5)     // 最大空闲连接数
   ```

3. **错误处理**
   ```go
   if err != nil {
       return nil, fmt.Errorf("操作失败: %w", err)  // 使用 %w 包装错误
   }
   ```

### 4.4 数据库层

#### 数据库连接

```go
// mysql.go
func InitDB() error {
    // 1. 从环境变量读取配置
    dsn := fmt.Sprintf("%s:%s@tcp(%s:%s)/%s?charset=utf8mb4&parseTime=True&loc=Local",
        os.Getenv("DB_USER"),    // 用户名
        os.Getenv("DB_PASS"),    // 密码
        os.Getenv("DB_HOST"),    // 主机
        os.Getenv("DB_PORT"),    // 端口
        os.Getenv("DB_NAME"),    // 数据库名
    )

    // 2. 初始化数据库连接
    DB, err = sql.Open("mysql", dsn)
    if err != nil {
        return fmt.Errorf("数据库连接错误:%w", err)
    }

    // 3. 测试连接
    if err = DB.Ping(); err != nil {
        return fmt.Errorf("数据库连接失败:%w", err)
    }

    // 4. 设置连接池
    DB.SetMaxOpenConns(25)    // 最大打开连接数
    DB.SetMaxIdleConns(5)     // 最大空闲连接数

    return nil
}
```

### 4.6 中间件

#### CORS 跨域中间件

```go
// auth.go
func CORS(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // 设置 CORS 响应头
        w.Header().Set("Access-Control-Allow-Origin", "*")
        w.Header().Set("Access-Control-Allow-Methods", "GET,POST,OPTIONS")
        w.Header().Set("Access-Control-Allow-Headers", "Content-Type,Authorization")

        // 处理 OPTIONS 预检请求
        if r.Method == "OPTIONS" {
            w.WriteHeader(http.StatusNoContent)
            return
        }

        next.ServeHTTP(w, r)
    })
}
```

#### 使用方式

```go
// main.go
func main() {
    mux := http.NewServeMux()
    handler.RegisterUserRoutes(mux, userSvc)

}
```

```sql
    email VARCHAR(100) UNIQUE,
    avatar_url VARCHAR(255) DEFAULT '',
```

## 9. 常见问题

### 9.1 数据库连接失败

**错误**：`Error 2002 (HY000): Can't connect to local MySQL server`

**解决方案**：
1. 检查 MySQL 服务是否启动：`sudo systemctl status mysql`
2. 检查 `.env` 配置是否正确
3. 检查 MySQL 用户权限

### 9.2 端口被占用

**错误**：`bind: address already in use`

**解决方案**：
```bash
# 查看占用端口的进程
lsof -i :8899

# 结束进程
kill -9 <PID>

# 或修改端口
PORT=9000 go run cmd/server/main.go
```

### 9.3 模板文件找不到

**错误**：`open web/templates/login.html: no such file or directory`

**解决方案**：
1. 确保模板文件路径正确
2. 检查 `web/` 目录是否存在
3. 运行 `go run cmd/server/main.go` 时在项目根目录

### 9.4 密码未加密

**当前状态**：密码明文存储

**解决方案**：
```go
import "golang.org/x/crypto/bcrypt"

// 加密密码
hashedPassword, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
user.Password = string(hashedPassword)

// 验证密码
err := bcrypt.CompareHashAndPassword([]byte(hashedPassword), []byte(inputPassword))
```

### 9.5 模板渲染错误

**错误**：`html/template: template: :1: function "html" not defined`

**解决方案**：
在 HTML 中使用 `{{ . }}` 时，Go 会自动转义 HTML。如需显示 HTML，使用：
```html
{{ .Title }}  <!-- 自动转义 -->
{{ .Title | html }}  <!-- 不转义，可能存在 XSS 风险 -->
```
